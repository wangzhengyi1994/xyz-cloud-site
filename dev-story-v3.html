02 — DESIGN CONSTRAINTS

给 AI 立的规矩

AI 不缺能力，缺的是约束。没有规范的 AI 输出就是"AI 味儿"——以下 6 条规则让它老实干活。

只用令牌，禁止硬编码

所有颜色、间距、字号必须引用 CSS 变量，写死 #4362ff 直接打回

字重只有两档

300 和 500，不允许出现 400 / 600 / 700。AI 最爱加粗，一律驳回

间距走 4px 网格

8 / 16 / 24 / 32 / 48 / 80，禁止出现 15px、30px 这种"差不多"的值

深色背景文字层级

标题 ≥ white-8，正文 white-5，辅助 white-3。AI 默认全白，必须分层

动效要有物理感

cubic-bezier 曲线，非 linear。入场 0.6s ease，hover 0.3s。禁止弹跳、果冻等花哨动画

一个文件，零依赖

不许引入 UI 框架、不许拆文件、不许 npm install。约束反而逼出更干净的代码

03 — AI BOUNDARY

AI 目前还做不到的事

AI 能写代码、能排版，但这些领域仍然需要设计师的判断力和专业工具。
不知道这些边界在哪——恰恰说明你还没到能判断的水平。

视觉设计

2.5D 等距风格插画

AI 生成的等距图透视永远"差一点"——角度不统一、光影不一致、细节经不起放大。这类风格图仍然依赖手工绘制和严格的透视网格。

3D 建模

精确三维模型

品牌级 3D 资产需要精确拓扑、干净布线、正确 UV。AI 生成的模型面数爆炸、布线混乱，导入引擎后根本没法用。

Rive Runtime
交互动效

状态驱动的复杂动效

Lottie 只能播放线性时间轴。真正的交互动效——响应输入、切换状态、物理反馈——我们用 Rive 做，AI 写不出这种状态机逻辑。

06 — CHALLENGES & DECISIONS

踩坑与决策

每一个"回退"都是快速实验和理性判断。

Problem

地球上台湾消失

低精度 TopoJSON + 大 step，台湾没有陆地点。step=1 有 65k 点太卡。

Solution

高精度 + 手动补点

countries-50m.json + step=2（~16k）+ 手动补台湾/海南/斯里兰卡。

Problem

点阵 SVG 性能崩溃

数千 rect 用 CSS Transform 缩放，浏览器重排卡顿。

Solution

Canvas + 涟漪算法

DOMParser 提取坐标 → Canvas 逐帧。涟漪 12px/帧 + smoothstep 放大镜。

Problem

Core Value 暗色失败

立方体和文本在暗色背景上对比度不好。

Solution

4 分钟果断回退

提交后立即还原，CLAUDE.md 记录"确认保持白色"。

Problem

Cosmic Spectrum 喧宾夺主

GSAP 全屏滚动光谱完全抢走注意力。

Solution

大胆砍掉

display:none 观察 → 确认不需要 → 彻底删除 + 移除 GSAP CDN。

12 — LESSONS LEARNED

复盘与收获

一个设计师，用 AI 把脑子里的特效落地成代码。这条路上踩过的坑和想明白的事。

01

设计师最大的痛：看得到，做不出

脑子里清楚想要 5000 粒子变形、地球飞线、数据隧道的效果，Figma 里也画得出概念稿——但打开编辑器就卡住了。Three.js 文档看不懂，矩阵运算写不来。AI 之前，这些特效只能等开发排期，等到的还不一定是你想要的。

02

AI 让"不好看，再改"成为可能

以前改一个特效参数要提需求、排期、联调。现在是：看一眼→"粒子太稀了"→2 分钟后新版本出来→"颜色偏了"→再 2 分钟。数据隧道 8 轮微调、点阵动画 7 次迭代——这种密度的打磨，以前根本不可能。

03

审美判断是人不可替代的价值

AI 能写出 Three.js 代码，但它不知道"这个动画看着不舒服"。Core Value 暗色背景 4 分钟回退、Cosmic Spectrum 果断砍掉——这些决策只有设计师能做。代码可以生成，品味不行。

04

不需要懂代码，但必须懂体验

不懂代码怎么实现，但知道怎么做出来效果好、性能好。高级审美和交互直觉才是驱动 AI 输出高质量结果的真正引擎——你要能精准描述"哪里不对"，AI 才知道往哪改。

05

2 天做完不是重点，一个人做完才是

以前这种项目：设计 → 开发 → 联调 → 改需求 → 返工，最少两周。现在设计师自己闭环——从 Figma 令牌到 Three.js 粒子，从构思到上线，中间没有任何沟通损耗。
